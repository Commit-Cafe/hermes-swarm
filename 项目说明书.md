# Hermes Swarm

# 多 Hermes-Agent 可视化管理工作项目 — 项目说明书 v0.1

---

## 一、项目概述

Hermes Swarm 是一个**多 Hermes-Agent 并行任务编排与可视化监控平台**。它允许用户同时启动和管理多个 Hermes Agent 实例，实时监控运行状态，自动收集执行结果，并追踪 token/cost 消耗。

本项目定位为**个人效率工具 + 可展示项目**，复用了开源项目 OAIHUB（MIT 许可证）的前端仪表盘组件，后端为自研的轻量级进程管理器。

---

## 二、技术架构

| 层级 | 技术选型 |
|------|----------|
| 后端 | FastAPI + aiosqlite + SSE |
| 前端 | Next.js 15 + React 19 + shadcn/ui + Recharts + Tailwind CSS |
| 存储 | SQLite（文件级持久化，`data/hermes_swarm.db`） |
| 进程管理 | Python asyncio.subprocess → WSL2 → hermes CLI |
| 实时通信 | SSE (Server-Sent Events) |
| 项目路径 | `D:\trae_Dproject2\hermes-swarm` |

## 八、已知限制与风险

1. **Hermes 进程管理依赖 WSL2 环境**，Windows 直接运行不支持
2. **进程管理器为内存级单例**，后端重启后运行中任务会丢失追踪（SQLite 持久化保留历史记录）
3. **最大并发数默认为 4**（可通过环境变量 `MAX_CONCURRENT_AGENTS` 调整）
4. **Token 计数和成本追踪功能尚未实现**（需解析 Hermes 输出流）
5. **前端 SSE 对接尚未完成**，当前状态更新依赖定时轮询（3-5 秒间隔）

## 十、环境变量参考

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| `API_HOST` | `0.0.0.0` | 后端监听地址 |
| `API_PORT` | `8001` | 后端端口 |
| `CORS_ORIGINS` | `localhost:3000,3001,5173` | 允许的前端跨域地址 |

| `NEXT_PUBLIC_API_URL` | `http://localhost:8001/api/v1` | 前端调用的后端 API 地址 |
